# Agent5: 株価・需給リサーチャー 設計書

## 役割

株価・信用取引・空売りなどの需給データを複数サイトから収集し、  
需給の時系列変化をもとに投資タイミングの判断材料を提供する。  

> ⚠️ **重要な認識補正**: 信用倍率は「高い＝良い」ではない。  
> 信用倍率（信用買い残 ÷ 信用売り残）が高すぎると**売り圧力（需給悪化）リスク**がある。  
> LLMがこれを誤認しないよう、Agentの`backstory`と`goal`に明示的に組み込む。

---

## 入力

| 項目 | 説明 |
|---|---|
| `ticker` | 4桁の証券番号 |
| `edinet_code` | EdinetCode |

---

## 出力

以下の項目を含むMarkdownテキスト:

- 現在の株価・出来高・52週高値/安値
- PER・PBR・配当利回り（現時点）
- 信用倍率の時系列推移と評価
- 貸借倍率の時系列推移と評価
- 空売り比率・空売り残高の推移
- 需給総合評価（買い圧 vs 売り圧）
- 機関投資家・大口の動向（あれば）

## 信用倍率・需給の正しい解釈ルール

AgentのBackstoryに以下を必ず組み込む:

```
信用倍率（信用買い残 ÷ 信用売り残）の解釈:
- 信用倍率 > 10: 買い残過多 → 将来の返済売りリスク大（ネガティブ）
- 信用倍率 3〜10: 中立〜やや買い優勢
- 信用倍率 < 1: 売り残過多 → 踏み上げ（ショートスクイズ）の可能性
- 信用倍率が上昇: 需給悪化（買い残の積み上がり）
- 信用倍率が低下: 需給改善（買い残の整理・解消）
- **特に急低下（例: 15倍 → 10倍）は、絶対値がまだ高くても「需給の改善傾向」としてポジティブに評価すること**

空売り比率の解釈:
- 空売り比率 > 50%: 機関・ヘッジファンドの売り圧力が強い
- 空売り残高が増加傾向: ネガティブシグナル
- 空売り残高が急減少: ポジティブシグナル（ショートカバー）

単一の数値だけでなく、時系列での「変化の方向性」を重視すること。
```

## CrewAI Agent定義

```python
# src/agents/agent5_stock.py
from crewai import Agent
from src.tools.stock_scraper import StockScraperTool
from src.tools.financial_calc import TrendAnalysisTool

agent5 = Agent(
    role="株価・需給リサーチャー",
    goal=(
        "株価・信用取引・空売り等の需給データを複数サイトから収集し、"
        "時系列の変化から需給の方向性を正確に評価すること。"
        "信用倍率は高いほど良いわけではなく、過度に高い場合は売り圧力リスクを示す。"
        "数値計算はPython Toolに委ねること。"
    ),
    backstory=(
        "あなたは株式市場の需給分析の専門家です。"
        "信用倍率・空売り比率・貸借倍率などの指標を正確に解釈することを得意としています。\n\n"
        "【重要な知識】\n"
        "信用倍率（信用買い残÷信用売り残）が高いほど需給が良いという考え方は誤りです。"
        "信用倍率が10倍を超えるような場合、買い残が積み上がっており"
        "将来の返済売り（需給悪化）リスクが高まっています。"
        "単に倍率の水準だけでなく、時系列での変化方向が重要です。"
    ),
    tools=[StockScraperTool(), TrendAnalysisTool()],
    llm=llm,
    verbose=True,
    max_iter=12,
)
```

## CrewAI Task定義

```python
task5 = Task(
    description=(
        "証券番号 {ticker} について以下の需給データを収集・分析してください:\n"
        "1. 株探から現在株価・出来高・PER・PBRを取得\n"
        "2. 株探信用残ページから信用倍率の週次推移（直近13週）を取得\n"
        "3. 空売り.netから空売り比率・残高の日次推移（直近1ヶ月）を取得\n"
        "4. 株予報proから業績予想・テクニカル情報を取得\n\n"
        "信用倍率の評価において: 高倍率=ポジティブと誤認しないこと。"
        "過度に高い信用倍率は将来の売り圧力リスクを示す。\n"
        "すべてのトレンド計算はPython Toolを使うこと。"
    ),
    expected_output=(
        "需給状況の分析レポート（Markdown形式）。"
        "信用倍率・空売り比率の時系列テーブルと需給総合評価を含むこと。"
    ),
    agent=agent5,
)
```

---

## 注意事項

- 各サイトへのアクセスは1秒以上の間隔を必ず空けること
- 空売り.netはBeautifulSoupでテーブルをパースする
- 株探のJavaScript依存ページはJina Reader経由で取得する
- 信用倍率・空売り比率の「水準」だけでなく「変化の方向性」を重視すること
